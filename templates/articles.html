Beginner
01. What is Phishing?

Phishing is a social engineering technique where attackers impersonate trustworthy entities to deceive victims into handing over credentials, financial details, or executing malware. It forms the entry vector for 90% of data breaches.

  • Deceptive Salutation: Vague greetings like "Valued Customer".
  • Sense of Urgency: Threatening immediate suspension.
  • Mismatched Domain: Subtle typos in host names.
Beginner
02. Password Security Guide

Weak passwords are easy to crack via brute-force or credential stuffing. A resilient password strategy requires high entropy and modern authentication layers to withstand dictionary attacks.

  • Length Over Complexity: Use passphrases (4+ random words).
  • Multi-Factor Auth: Enable authenticator apps (TOTP), avoiding SMS.
  • Credential Vaults: Implement password managers to generate distinct credentials.
Beginner
03. Email Security Awareness

SMTP was not designed with security in mind, allowing sender spoofing. Identifying header anomalies and envelope signatures is crucial for preventing malicious inbox entries.

  • SPF / DKIM Checks: Inspect authentication validation records.
  • Display Name Spoofing: Double-check the sender's actual email, not just the name.
  • Attachment Risk: Never launch macros in incoming documents (.docm, .vbs).
Intermediate
04. Types of Phishing Attacks

Phishing has evolved from bulk spam to highly targeted attacks utilizing open-source intelligence (OSINT). Understanding different variants is crucial to stopping targeted intrusions.

  • Spear Phishing: Highly customized campaigns targeting specific individuals.
  • Whaling: Spear phishing directed at high-level executives (C-Suite).
  • Vishing & Smishing: Attacks carried out via Voice Calls or SMS text alerts.
Intermediate
05. Secure Browsing Practices

Browsers are our gateway to the web, exposing users to cross-site scripting (XSS), drive-by downloads, and domain hijackings. Secure client-side settings mitigate these risks.

  • HTTPS-Only Mode: Auto-upgrades HTTP connections to encrypted tunnels.
  • Secure DNS: Set up DNS-over-HTTPS (DoH) via Cloudflare or Google.
  • Active Script Control: Block automatic popups and unauthorized extensions.
Intermediate
06. Real-World Case Studies

Analyzing prominent corporate breaches proves that social engineering overrides technical controls. Here are key historical lessons:

  • Google & Facebook (2013): A lone actor spoofed hardware vendor invoice emails, stealing $100M.
  • FACC (2016): A whaling attack impersonated the CEO requesting an acquisition transfer, costing $54M.
  • Target Corporation (2013): Compromised via HVAC contractor phishing credentials, breaching 40M cards.

07. AI Phishing Detection Process

Static signature matching fails against dynamic URL generation. CyberShield utilizes machine learning to parse lexical tokens and estimate risk values instantly.

  • Feature Vectorization: Extracting length, IP usage, and redirect signatures.
  • Random Forest Classification: Processing vectors through 100 decision trees.
  • Probability Score: Mapping decision leaf votes to confidence percentage.
Advanced
08. Cyber Kill Chain Overview

Developed by Lockheed Martin, this structured framework maps the phases of a cyber attack from initial reconnaissance to goal completion. Disrupting any phase breaks the chain.

  • Reconnaissance & Weaponization: Gathering OSINT and building payloads.
  • Delivery & Exploitation: Transmitting payloads (typically via Phishing URLs).
  • Command & Control: Establishing remote access for data exfiltration.
Advanced
09. OWASP Top 10 Overview

The Open Web Application Security Project (OWASP) indexes the top ten most critical web app vulnerabilities, establishing defensive coding practices worldwide.

  • A01: Broken Access Control: Bypassing authorization tokens.
  • A03: Injection (SQLi/XSS): Executing untrusted command strings.
  • A05: Security Misconfiguration: Leaving default settings or debug codes open.

Blueprints of Common Phishing Email Scams

Analyze real-world spoofing email structures and learn how to identify their underlying indicators.

Case 1: Account Lockout Spoof High Risk
From: security-alerts@pay-pal-verification.net
Subject: URGENT: Access restricted on your card wallet!

Dear Customer,

We noticed anomalous access signatures on your profile wallet. To restore access, execute validation of identity details in the link below:
http://paypal-verification-portal.com/update

PayPal Security Department
Red Flags: Spoofed domain sender address (net vs com), insecure HTTP protocol URL, generic customer greeting, high emotional threat pressure.
Case 2: Corporate SSO Expiring High Risk
From: it-sso-portal@corp-domain-portal-security.net
Subject: Required: Password Expiry SSO Synchronize

Hello Employee,

Your corporate single-sign-on (SSO) credential expires in 12 hours. Synch credentials to maintain access to corporate mailboxes:
https://bit.ly/3yRst5A

SSO Security Helpdesk
Red Flags: Sender domain combination names (domain-portal-security), URL shorteners (bit.ly) obfuscating real destination, immediate temporal restriction pressure.
